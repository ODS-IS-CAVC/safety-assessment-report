# SCT算出ツール

Safety Cushion Time 算出・可視化ツール

---

## SCTツールとは

**Safety Cushion Time (SCT)** を自動計算・可視化するデスクトップアプリケーション

### 主な機能

- OpenDRIVE道路データの読み込み・表示
- 車両軌跡データ（CSV）の読み込み
- 車両間の相対距離・相対速度の自動計算
- SCT（縦方向・横方向）の時系列グラフ表示
- 動画再生機能による時間軸上の可視化

---

## SCTとは

**Safety Cushion Time（安全余裕時間）**

車両間の衝突余裕を時間で表す安全指標

### 計算式

- **縦方向SCT** = dx（進行方向の相対距離） / vx（進行方向の相対速度）
- **横方向SCT** = dy（左右方向の相対距離） / vy（左右方向の相対速度）

### 符号規則

- **正の値**: 対象車両が自車に接近中
- **負の値**: 対象車両が自車から遠ざかる

---

## 必要な入力データ

### 1. OpenDRIVEファイル（.xodr）

道路・車線の線形情報

- 道路ネットワーク
- 車線情報
- ジオメトリ（直線、円弧、クロソイドなど）

### 2. 車両軌跡CSVファイル

自車両および対象車両の時系列位置データ

---

## CSVフォーマット（必須列）

### 必須列

```
timestamp, pos_x, pos_y, yaw_rad, vel_x, vel_y
```

### 例

```csv
timestamp,pos_x,pos_y,pos_z,yaw_rad,roll_rad,pitch_rad,vel_x,vel_y,vel_z
0.0,-2147.682,-53.852,9.033,0.0045,-0.0094,-0.0067,18.588,0.085,0.245
0.01,-2147.497,-53.851,9.037,0.0045,-0.0094,-0.0058,18.497,0.087,0.421
```

### ファイル命名規則

ファイル名に特定の制約はありません。ただし、以下の命名規則に従うと、車両サイズが自動設定されます。

#### 推奨形式

- `*_Veh_車両名.csv` または `*_Tgt_車両名.csv`
- 例: `CarMaker_Veh_HinoProfia_self.csv`、`scenario_1_divp_Tgt_Corolla.csv`

この形式に従うと、ファイル名から車両名（例: `HinoProfia`、`Corolla`）が自動抽出され、`vehicle_bbox.json`に登録されている場合は車両サイズ（BoundingBox）が自動設定されます。

命名規則に従わない場合は、手動でBoundingBox設定が必要です。

#### 補足

`_Veh_`や`_Tgt_`が含まれていても、読み込み時にどちらを自車両/対象車両とするかはユーザーが選択できます。

## 起動方法

### Windows

1. `sct-tool.exe` をダブルクリック
2. またはインストーラー（.msi / .exe）からインストール後、スタートメニューから起動

### 初回起動時

- WebView2ランタイムが自動インストールされる場合があります（Windows 10）

---

## データ読み込み手順

### ステップ1: OpenDRIVEファイルの読み込み

1. メニュー「ファイル」→「OpenDRIVEを読み込む」
2. `.xodr` ファイルを選択
3. 道路ネットワークが表示される

### ステップ2: 自車両軌跡の読み込み

1. メニュー「ファイル」→「自車両軌跡を読み込む」
2. 自車両として使用するCSVファイルを選択
3. 自車両が赤色で表示される

---

## データ読み込み手順（続き）

### ステップ3: 対象車両軌跡の読み込み

1. メニュー「ファイル」→「対象車両軌跡を読み込む」
2. 対象車両として使用するCSVファイルを選択
3. 対象車両が青色で表示される
4. SCTグラフが自動生成される

---

## 車両サイズ（BoundingBox）の設定

### 概要

車両のBoundingBoxは、`vehicle_bbox.json`による自動設定のみサポートしています。ファイル名が命名規則に従っている場合、車両名が自動抽出され、登録済みの車両情報からBoundingBoxが設定されます。

**注**: 現在、UI上での手動設定機能は実装されていません。

### ファイルの場所

- **実行ファイルと同じディレクトリ**: `sct-tool.exe`と同じ場所に配置
- **開発環境**: `docs/vehicle_bbox.json`

### JSON形式

```json
{
  "vehicles": [
    {
      "name": "車両名",
      "BoundingBox": {
        "Center": {
          "x": 車両中心のX座標（m）,
          "y": 車両中心のY座標（m）,
          "z": 車両中心のZ座標（m）
        },
        "Dimensions": {
          "height": 車両の高さ（m）,
          "length": 車両の長さ（m）,
          "width": 車両の幅（m）
        }
      }
    }
  ]
}
```

### 定義例

```json
{
  "vehicles": [
    {
      "name": "HinoProfia",
      "BoundingBox": {
        "Center": {
          "x": 2.536,
          "y": 0.0,
          "z": 1.849
        },
        "Dimensions": {
          "height": 3.698,
          "length": 11.87,
          "width": 2.564
        }
      }
    },
    {
      "name": "Corolla",
      "BoundingBox": {
        "Center": {
          "x": 1.286,
          "y": 0.0,
          "z": 0.718
        },
        "Dimensions": {
          "height": 1.435,
          "length": 4.495,
          "width": 1.745
        }
      }
    }
  ]
}
```

### 車両名の抽出規則

ファイル名から車両名を抽出する際の規則：

1. `*_Veh_車両名*.csv` → `車両名`部分をアンダースコアで区切り、最初の部分を使用
   - 例: `CarMaker_Veh_HinoProfia_self.csv` → 車両名 = `HinoProfia`
2. `*_Tgt_車両名*.csv` → 同様に抽出
   - 例: `scenario_1_Tgt_Corolla.csv` → 車両名 = `Corolla`
3. 抽出した車両名が`vehicle_bbox.json`の`name`フィールドと一致すれば、BoundingBoxが自動設定されます

### 座標系

- **Center**: 車両の後軸中央を基準とした車両中心の相対位置（SDMG形式準拠）
- **Dimensions**: 車両の外形寸法
- 単位: すべてメートル（m）

### 注意事項

- 車両名が`vehicle_bbox.json`に登録されていない場合、エラーが発生します
- その場合は、手動でBoundingBox設定を行うか、`vehicle_bbox.json`に車両を追加してください

### 3つの主要エリア

1. **レンダリング領域**（上部）

## SCTグラフの見方

### 2つのグラフ

1. **縦方向SCT（SCT_x）**
   - 進行方向の安全余裕時間
   - 正: 対象車両が前方から接近
   - 負: 対象車両が後方へ遠ざかる

2. **横方向SCT（SCT_y）**
   - 左右方向の安全余裕時間
   - 正: 対象車両が横から接近
   - 負: 対象車両が横へ遠ざかる

### グラフの色

- **青線**: SCT値
- **赤い縦線**: 現在時刻

---

## SCT値の解釈

### 安全性の判断基準

- **SCT > 3.0秒**: 安全
- **1.0秒 < SCT < 3.0秒**: 注意
- **SCT < 1.0秒**: 危険

### 特殊なケース

- **SCT = ∞ (無限大)**: 相対速度が0（並走状態）
- **SCT = NaN**: データ不足または初期化中
- **SCT < 0**: 対象車両が遠ざかる（衝突リスクなし）

## トラブルシューティング

### アプリが起動しない

- **原因**: WebView2ランタイムが未インストール
- **解決策**: Windows Updateを実行、または手動でWebView2をインストール

### CSVファイルが読み込めない

- **原因**: フォーマット不正、必須列の欠落
- **解決策**: ヘッダー行に `timestamp, pos_x, pos_y` が含まれているか確認

### グラフが表示されない

- **原因**: 対象車両が読み込まれていない
- **解決策**: 自車両と対象車両の両方を読み込む

---

## トラブルシューティング（続き）

### 車両が表示されない

- **原因**: 座標系のずれ、OpenDRIVEと軌跡データの不一致
- **解決策**: ズームアウトして広範囲を確認、座標値をチェック

### 動作が重い

- **原因**: 大量の軌跡データ
- **解決策**: データを間引く、PCのメモリを増やす

### ビルドエラー

- **原因**: 依存関係の不足、環境設定の問題
- **解決策**: `npm install` を再実行、Visual Studio Build Toolsを確認
